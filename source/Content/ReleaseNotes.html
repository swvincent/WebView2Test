May 14, 2026 v3.2.0
DataForge 3.2 — Streaming Pipelines Latest
@mcarvalho released this · Compare: v3.1.1...v3.2.0

🚀 What's New

  • Added streaming pipeline support — process records without buffering entire datasets in memory (@mcarvalho)
  • New Pipeline.FromStream<T>() factory method with built-in backpressure handling
  • Introduced parallel fan-out stages via pipeline.FanOut(n) for CPU-bound transformations (@jsullivan)
  • Added IAsyncEnumerable<T> source and sink adapters
  • New DataForge.Metrics package exposing OpenTelemetry counters for pipeline throughput and latency
  • JSON source now supports JSON Lines (NDJSON) format in addition to arrays (@priya_k)

🐛 Bug Fixes

  • Fixed a race condition in BatchStage that could drop the final partial batch on pipeline completion (@jsullivan)
  • Resolved ObjectDisposedException when cancelling a pipeline mid-flush
  • Fixed CSV parser mishandling quoted fields that span multiple lines
  • Corrected DateTimeOffset timezone preservation when writing Parquet output (@alin_d)

⚡ Performance

  • Reduced allocation pressure in the hot path by ~40% via ArrayPool<T> in sort stages
  • Parquet writer throughput improved by 25% on .NET 10 through vectorized column encoding
// New streaming usage
await Pipeline
    .FromStream<Order>(sourceStream)
    .Filter(o => o.Total > 100m)
    .FanOut(4, stage => stage.Transform(Enrich))
    .WriteTo(sink)
    .RunAsync(cancellationToken);

Apr 3, 2026 v3.1.0
DataForge 3.1 — Schema Evolution & Excel Support Feature Breaking
@mcarvalho released this · Compare: v3.0.2...v3.1.0

🚀 What's New

  • Added schema evolution support — pipelines can now handle source columns being added, removed, or reordered without failing (@mcarvalho)
  • New Excel source (ExcelSource) supporting .xlsx and .xls via a built-in reader — no Office interop required (@alin_d)
  • Introduced pipeline.Checkpoint(path) for fault-tolerant pipelines — resumes from last committed offset on restart
  • Added LookupJoin stage for enriching records from an in-memory dictionary with O(1) lookups
  • New DataForge.Testing package with FakeSource<T> and assertion helpers for unit-testing pipeline logic (@jsullivan)

💥 Breaking Changes

  • IStage<TIn, TOut> now requires implementing ValueTask ProcessAsync instead of Task ProcessAsync — update custom stages accordingly
  • PipelineOptions.MaxDegreeOfParallelism renamed to PipelineOptions.Concurrency for consistency with the new fan-out API
  • Removed deprecated Pipeline.Run() synchronous overload — use await pipeline.RunAsync() instead
// Migration: custom stages
// Before (3.0)
public Task ProcessAsync(TIn input, IOutputWriter<TOut> writer) { ... }

// After (3.1)
public ValueTask ProcessAsync(TIn input, IOutputWriter<TOut> writer) { ... }

🐛 Bug Fixes

  • Fixed Parquet decimal precision loss for values with more than 18 significant digits (@priya_k)
  • Resolved a deadlock when using pipeline.Merge() with two sources that produce at different rates
